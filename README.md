# router
A cost-performance adaptive routing method for large language models is investigated for multi-task application scenarios.
In view of the high
invocation cost of large language models and the differences among task requirements, task type, sample complexity, task-model profiles and runtime availability are jointly
considered to select execution models from a heterogeneous commercial model pool. A unified multi-task data interface, a task-model profiling mechanism and a
rule-enhanced statistical routing strategy are designed. Online evaluation and offline
budget-accuracy analysis are carried out under the same experimental protocol. The
results show that the proposed method can support model selection and result analysis
in real API invocation scenarios, and provides an implementable reference for cost
control and performance tradeoff in multi-model deployment.
